# Как агенты делят работу

Правило одно: **зона захватывается до правки, а не после конфликта.**

Пересечение задач вычисляет программа, а не человек и не догадка агента.

---

## 1. Почему так, а не «договоритесь между собой»

- `public/app.js` — около 200 КБ одним файлом. Восемь агентов правят его одновременно.
- Агенты живут в **разных процессах**, а иногда и в **разных сессиях**. Общей памяти нет,
  промпт соседа невидим. Договориться нечем.
- Правило в промпте модель размывает за 5-6 обменов. Проверено на этом же проекте:
  гейт против учебной повестки в чате пришлось выносить в код по той же причине.
- 23.07 параллельная сессия унесла незакоммиченные правки чужого агента в свой коммит.
  Никто не был виноват — не было механизма.

Вывод: нужен **запрет**, а не просьба. Запрет живёт в реестре захватов и в хуке.

---

## 2. Порядок работы агента

```bash
# 1. Посмотреть, что занято
node scripts/claim.mjs status

# 2. Захватить свою зону. Команда САМА покажет конфликт и не даст захватить занятое.
node scripts/claim.mjs claim \
  --who "designer" \
  --task "Гримёрка: портрет и непрочитанное" \
  --paths "public/app.js#renderCabinet,public/style.css:200-330"

# 3. Работать. Долгие задачи — продлевать, иначе захват протухнет через 45 минут.
node scripts/claim.mjs beat --who "designer"

# 4. Освободить, когда закончил
node scripts/claim.mjs release --who "designer"
```

Код возврата `claim` и `check`: **0 — свободно, 1 — конфликт**. Конфликт печатает,
кто держит зону и над чем работает.

**Захват не удался — не правь.** Три законных выхода: сузить свою зону, взять другую
задачу, дождаться освобождения. Четвёртого нет.

---

## 3. Формат зоны

| Запись | Что значит |
|---|---|
| `public/app.js` | файл целиком |
| `public/app.js#renderCabinet` | именованный участок |
| `public/app.js:800-950` | диапазон строк |

Правила пересечения:
- **файл целиком бьётся с любым участком в нём** — поэтому в общих файлах бери участок, не файл;
- одинаковые якоря бьются, разные — нет;
- диапазоны строк бьются по факту перекрытия;
- якорь против диапазона сопоставить нельзя — считается конфликтом. Это осознанно:
  лучше лишний раз спросить, чем молча испортить.

## 4. Страж

`scripts/claim-guard.mjs` висит на `PreToolUse` и **отклоняет** правку файла из списка
охраняемых, если живого захвата нет.

Охраняются: `public/app.js`, `public/style.css`, `public/index.html`,
`public/curriculum.js`, `public/songs.js`, `lib/reply.js`, `api/song.js`,
`api/chat.js`, `api/bot.js`. Остальное не сторожим — страж должен мешать только там,
где реально бьются.

Агент объявляет себя переменной `CLAIM_WHO`. Без неё страж пропускает правку, но
предупреждает, чей это участок.

**Хуки читаются при старте сессии.** Изменил `settings.local.json` — перезапусти сессию,
иначе страж не поднимется. Проверено 24.07: правка на лету не подхватывается.

Отладка: `CLAIM_DEBUG=1` печатает, что страж увидел. Обход на один вызов: `CLAIM_GUARD=off`.

## 5. Правила, которые не проверяются программой

Их всё равно соблюдать.

1. **`Write` на охраняемый файл запрещён всегда.** Только `Edit` по уникальным якорям.
   Полная перезапись затирает чужую работу целиком, и захват от этого не спасает.
2. **Перед каждым `Edit` перечитывай кусок.** Файл меняется под тобой.
3. **Не коммить и не пушить.** Сводит ведущий. Иначе унесёшь в свой коммит чужие правки.
4. **Стык двух зон — через файл-контракт.** Кто отдаёт данные, описывает формат в
   `IDOLINGO_CONTRACT_<тема>.md` **до** написания кода, чтобы сосед не ждал.
   Так уже сделано для песни-тетради и тетради-озвучки, механика рабочая.
5. **Длинная работа — короткими подходами.** Сделал шаг, проверил, записал в свой
   журнал `review/.../<ТЕМА>_NOTES.md`. Обрывы связи случаются: 24.07 их было пять.
   С журналом падение стоит минуты, без журнала — всей работы.

---

## 6. Что это не чинит

Реестр разводит агентов, но **не устраняет причину**: `public/app.js` слишком велик,
и участки в нём приходится размечать вручную.

Настоящее лечение — разрезать `app.js` на модули по экранам: кабинет, урок, песня,
тетрадь, чат, лендинг. Тогда зона агента совпадёт с файлом, пересечения станут
физически невозможны, а реестр останется нужен только для общих файлов.

Пока этого нет, реестр обязателен.
