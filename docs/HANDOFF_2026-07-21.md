## 2. Как устроен чат (важно для интеллекта)

- **Единый тред.** Все сообщения (веб + Discord + LINE + Telegram) пишутся в ОДНУ таблицу
  `chat_messages` по `idol_id`, без фильтра по каналу. Айдол при каждом ответе читает всю
  историю целиком → память общая и сквозная между сайтом и мессенджером в обе стороны.
- **Ядро ответа — `lib/reply.js`** (единый источник правды для всех каналов):
  - `buildPersona(idol, lang)` — собирает system-промпт: кто айдол, язык общения, правила
    подачи (живой друг, без методичных сносок, не объявляет себя ИИ), уровень корейского
    (`levelGuide`), близость 존댓말/반말 (`closenessGuide`), разбор песен.
  - `generateReply({db, idol, text, lang, channel})` — пишет входящее, тянет историю (40 посл.),
    строит messages-массив с ролями user/assistant, зовёт LLM, санитайзит, пишет ответ.
  - `generateProactive({db, idol, lang, reason, channel})` — айдол пишет ПЕРВОЙ (опенер по поводу).
- **Движок LLM:** fal.ai OpenRouter chat-completions (OpenAI-совместимый),
  модель `anthropic/claude-haiku-4.5`, URL `https://fal.run/openrouter/router/openai/v1/chat/completions`,
  auth `Key <FAL_KEY>`, ~$0.0006/вызов. **NB: старый `fal-ai/any-llm` мёртв (deprecated) — не использовать.**
  Формат: `{model, messages:[{role,content}], temperature, max_tokens}` → `{choices:[{message:{content}}]}`.

**Где живёт "интеллект" сейчас:** весь характер и поведение — в тексте `buildPersona()`.
Персонализации под конкретного юзера (его имя, интересы, стиль, слабые места в корейском)
пока НЕТ — персона строится только из карточки айдола + уровня. Это и есть точка роста.

## 5. Трек «Интеллект айдола» — с чего начинать

### Видение Сармата (дословно по смыслу)
Айдол сама подстраивается под характер юзера: задаёт ему вопросы, смотрит как отвечает,
понимает его психологию. Важная оговорка Сармата: **глубокое сближение происходит уже
В мессенджере, после переезда, а не на первых сообщениях на сайте** — на вебе задача лишь
зацепить и перетащить. Значит настоящую психо-адаптацию строим с прицелом на мессенджер-фазу.

### Что уже есть как «лёгкая версия»
Приглашение-момент (этап 4): айдол по своему чутью ловит момент позвать в мессенджер —
это первый пример, где айдол действует по ситуации, а не по скрипту. Тот же принцип
(LLM решает + код-рамки) можно масштабировать на весь характер.

### Возможные направления (не решено — обсудить с Сарматом)
1. **Профиль юзера / долгая память.** Извлекать и хранить факты о юзере (имя, интересы,
   любимая группа, слабые места в корейском, тон общения) в отдельной таблице; подмешивать
   в персону сжатой сводкой. Сейчас модель видит только последние 40 сообщений — старое забывается.
2. **Адаптация тона под характер.** Айдол считывает стиль юзера (застенчивый/бойкий, короткие/
   длинные сообщения) и подстраивает свою манеру.
3. **Проактивные вопросы «про него».** Айдол сама интересуется его жизнью, помнит ответы,
   возвращается к темам — строит ощущение, что реально знает человека.
4. **Модель посильнее для «глубоких» моментов.** Сейчас всё на Haiku 4.5 (дёшево). Для
   ключевых эмоциональных/психологических моментов можно точечно поднимать на Sonnet/Opus.
   Обсудить экономику (Haiku ~$0.0006 vs Sonnet/Opus дороже).

### Технические зацепки
- Расширять `buildPersona()` (добавить блок «что ты знаешь о друге: …»).
- Новая таблица `user_profile`/`user_memory` (миграция) под извлечённые факты.
- `HISTORY_LIMIT = 40` в `lib/reply.js` — предел «короткой» памяти; долгую делать отдельно.
- Стоимость: каждый чат-вызов ~$0.0006 (Haiku). Извлечение фактов = доп. вызовы, считать бюджет.

## 6. Как потрогать/протестить (шпаргалка)

- **Демо проактива (айдол пишет первой) прямо сейчас:**
  ```
  curl -X POST "https://k-pop-black.vercel.app/api/bot?action=proactive-tick&force=1&reason=streak&only=<idol_id>" \
    -H "x-ingest-secret: <INGEST_SECRET>"
  ```
  `force=1` минует пороги, `only=<idol_id>` — один айдол, `dry=1` — предпросмотр без записи.
  Тестовый айдол Сармата: `d15df4d7-0123-41c4-8ef9-cc48f2b51da2` (Discord Сармата привязан, lang ru).
- **Из локали доступны fal и Supabase напрямую** (Windows, Claude Code) — можно гонять
  LLM-вызовы и миграции через `pg` + `DATABASE_URL`. Ключи в `.env` (FAL_KEY, SUPABASE_*, DATABASE_URL, INGEST_SECRET, VERCEL_TOKEN).
- **Деплой:** push в `main` → Vercel автодеплой. Env-переменные пишутся через `VERCEL_TOKEN` (CLI).
- **Воркер апдейт:** `scp discord-worker/index.js root@67.205.175.124:/opt/idolingo-worker/ && ssh … systemctl restart idolingo-worker` (SSH-ключ `~/.ssh/idolingo_vps`; РФ→US рвётся на долгих операциях).
